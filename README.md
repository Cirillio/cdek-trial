# CDEK News Widget

Тестовое задание — универсальный компонент новостного блока для портала CDEK.

**Демо:** https://cdek-trial.vercel.app/

## Технологический стек

- **Core:** [React 19](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
- **Build Tool:** [Vite](https://vitejs.dev/)
- **Стилизация:** [Tailwind CSS v4](https://tailwindcss.com/)
- **Иконки:** [@tabler/icons-react](https://tabler.io/icons)
- **Фетчинг:** [@tanstack/react-query](https://tanstack.com/query)
- **Пакетный менеджер:** [Bun](https://bun.sh/)
- **Линтинг и форматирование:** ESLint + Prettier

## Запуск проекта

Убедитесь, что у вас установлен [Bun](https://bun.sh/).

1. **Установка зависимостей:**

    ```bash
    bun install
    ```

2. **Настройка окружения:**

    ```bash
    cp .env.example .env
    ```

    `.env.example` уже содержит рабочий URL тестового API — ничего менять не нужно.

3. **Запуск сервера разработки:**

    ```bash
    bun run dev

    bun run dev --host
    ```

    Проект будет доступен по адресу, указанному в терминале (обычно `http://localhost:5173`). Если проект запущен с флагом --host, то будет доступен внутри сети по адресу устройства (например, 192.168.0.11) с портом 5173.

4. **Сборка для production:**

    ```bash
    bun run build
    ```

5. **Предпросмотр production-сборки:**

    ```bash
    bun run preview
    ```

6. **Запуск линтера:**

    ```bash
    bun run lint
    ```

## Принятые решения

- **Стек:** Vite, react 19, ts strict, tailwind v4 (мой привычный стек, тайлик как родной уже)
- **Данные:** TanStack Query v5 — кэширование загруженных страниц (staleTime 5 мин), автоматическое обновление. До этого не работал с ним, но решил ознакомиться. Всё оказалось до боли понятно и удобно. В Nuxt есть буквально такие же хуки (useFetch, useAsyncData), поэтому освоился с этим за 5 минут.
- **Архитектура:** Выбрал FSD — `features/news`, `components`, `lib`, `types`, т.к. на прошлом проекте (о котором в анкете писал) использовали его и в целом мне он удобным показался.
- **Два типа карточек:** `standart` (горизонтальная) и `accent` (вертикальная с крупной обложкой). решил разделить на два компонента, т.к. посчитал, что возиться над условной версткой в зависимости от типа - нелучшее решение.
- **Пагинация:** управляется хуком `useNewsWidget`, кнопки блокируются на крайних страницах

## Реализованные дополнительные требования

- **Кэширование** — загруженные страницы не запрашиваются повторно (staleTime в react query)
- **Ленивая загрузка изображений** — компонент `LazyPicture` с skeleton-заглушкой и fallback при ошибке, разный (адаптивный) srcSet (s/m/l/hd)
- **Анимации** — простенькие анимации появления карточек (fade-in + slide-up), skeleton (pulse) при загрузке.
- **Типизация** - Использовал ТС, типизировав пропсы, хук и ответ от апи.

Пустые скелетоны на странице - просто wireframe для демонстрации как эти блоки новостей вписались бы на страницу.

Бейджа "Топ новость" нет, т.к. в ответе апи ни у одного объекта нет true у нужного поля.
